## Описание лабораторной работы
В рамках работы был разработан учебный плагин **USM Notes**, который добавляет на сайт раздел «Заметки» с возможностью присваивать каждой заметке **приоритет** и **дату напоминания**. Основные элементы плагина:

- **Custom Post Type (CPT)** – `usm_note` для заметок.
- **Пользовательская таксономия** – `priority` для приоритета (High, Medium, Low).
- **Метабокс** – поле выбора даты напоминания (`Reminder Date`) с валидацией.
- **Шорткод** – `[usm_notes]` для вывода заметок на страницах сайта с фильтрацией по приоритету и дате.

## Инструкции по запуску проекта

1. Скопировать папку плагина `usm-notes` в директорию `wp-content/plugins` локальной установки WordPress.
2. В файле `wp-config.php` включить отладку:
   ```php
   define('WP_DEBUG', true);

3. Зайти в **Админка → Плагины**, найти **USM Notes** и активировать.
4. Создать несколько заметок через **Notes → Add New**, заполняя:

   * Заголовок и текст
   * Приоритет (High / Medium / Low)
   * Дату напоминания (`Reminder Date`)
5. Создать страницу сайта (например, «All Notes») и вставить шорткод для отображения заметок.

## Краткая документация к плагину

### Custom Post Type

* **Название:** Notes
* **Поддержка:** title, editor, author, thumbnail
* **Архив:** есть
* **Иконка в админке:** dashicons-edit

### Таксономия

* **Название:** Priorities
* **Иерархическая:** Да
* **Слова для выбора:** High, Medium, Low

### Метабокс

* **Название поля:** Reminder Date
* **Тип:** HTML5 input type="date"
* **Валидация:** обязательное поле, дата не может быть в прошлом

### Шорткод

* `[usm_notes]` – вывод всех заметок
* `[usm_notes priority="high"]` – заметки с приоритетом High
* `[usm_notes before_date="2025-04-30"]` – заметки с датой напоминания до 30 апреля 2025
* `[usm_notes priority="high" before_date="2025-04-30"]` – комбинация фильтров

---

## Примеры использования плагина

1. **Вывод всех заметок на странице:**

   ```text
   [usm_notes]
   ```

2. **Вывод заметок с высоким приоритетом:**

   ```text
   [usm_notes priority="high"]
   ```

3. **Вывод заметок с датой напоминания до 30 апреля 2025:**

   ```text
   [usm_notes before_date="2025-04-30"]
   ```

4. **Комбинированный фильтр (High и до 30 апреля 2025):**

   ```text
   [usm_notes priority="high" before_date="2025-04-30"]
   ```

Если фильтры не соответствуют ни одной заметке, выводится сообщение:
**"Нет заметок с заданными параметрами"**

## Контрольные вопросы

1. **Чем пользовательская таксономия принципиально отличается от метаполя? Приведи пример, когда выбрать таксономию, а когда - метаданные**

Таксономия группирует записи по общему признаку, например, «Кухня» для рецептов. Метаполе хранит уникальные данные одной записи, например, «Время приготовления» рецепта.

2. **Зачем нужен nonce при сохранении метаполей и что произойдёт, если его не проверять?**

   Nonce защищает от CSRF — поддельных запросов на изменение данных. Если его не проверять, злоумышленник может изменить данные чужих записей без ведома пользователя.

3. **Какие аргументы register_post_type() и register_taxonomy() чаще всего важны для фронтенда и UX (назови минимум три и объясни почему)?**
* **`register_post_type()`**:

  * `public` — делает записи видимыми на сайте.
  * `has_archive` — включает архивную страницу для удобной навигации.
  * `rewrite` — задаёт красивые ЧПУ (URL) для лучшего UX и SEO.

* **`register_taxonomy()`**:

  * `hierarchical` — определяет, есть ли древовидная структура (категории) или плоская (теги), влияет на фильтры.
  * `show_in_rest` — делает таксономию доступной в редакторе Gutenberg и REST API.
  * `rewrite` — задаёт ЧПУ для таксономии, удобное отображение на фронтенде.
